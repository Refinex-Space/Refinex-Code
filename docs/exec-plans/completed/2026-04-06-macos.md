## Background

当前仓库只有 CLI 形态，缺少可承载后续聊天、配置、会话与本地集成能力的桌面应用入口；直接复用 Omni 完整桌面端会把 Rust 命令桥、模型设置、消息解析和会话存储一起带入，和当前 TypeScript 技术栈不匹配。

## Decision Log

| Date | Decision | Why |
| ---- | -------- | --- |
| 2026-04-06 | 新桌面端落在 `apps/desktop` 独立子应用，而不是把根仓库改造成 workspace | 保持当前 Bun CLI 工程稳定，降低对既有入口和依赖图的扰动 |
| 2026-04-06 | 用 Electron `main/preload` 替代 Omni 的 Tauri/Rust 边界 | 用户明确要求保留 TypeScript 技术栈，并抛弃 Rust 命令桥和设置/模型相关语义 |
| 2026-04-06 | 只迁移布局语言、主题、侧边栏、命令面板和终端面板，不迁移会话/设置/LLM 解析 | 最小壳层可先跑起来，同时避免把旧项目的业务耦合复制过来 |
| 2026-04-06 | 终端桥接首版使用 macOS `script` 分配 PTY，而不是引入 Electron 原生 PTY 模块 | 避免首个切片就处理 Electron ABI / 原生模块重编译问题，先满足可运行壳需求 |
| 2026-04-06 | Electron 二进制下载切到 `ELECTRON_MIRROR=https://npmmirror.com/mirrors/electron/` | 默认源在当前网络环境超时，镜像可达且能补齐本机运行时 |
